06.01 - List Data Structure and Iterables

    there is a 'linear' relationship to the elements of a collection. it is the same as the collection, but with 
        - elements of the list are indexed sequentially from zero to one less than the size of the list.
            - must define methods for adding, retrieving, changing, and removing elements from index, as well as method for determining index of element
            - must handle 'IndexOutOfBoundsException' if the index is invalid.

        - makes use of Java Iterable method, iterator that creates and return an Iterator object
        - Factory Methdod: method that creates and returns objects
        - Iterator object provides 3 operations: 
            - hasNext
            - next
            - remove

        - Array-based list - slow at removal and insertion, quick at reading a specifc index
        - LinkList-based   - fast at removal and insertion, slow at reading from specific index

        - the Iterator class has an anonymous inner class, the Anonymous class has no name, it is simply instantiated where needed
        - the Abstract class is built inside the Iterator object, and uses an anonymous class
            - iterator has hasNext(), and must utilize hasNext() before using next()
            - next() method
            - remove() method, must then shift all remaining element back 1 space -- this is O(N) complexity and is time consuming
