05.01 Array-Based Collections

    Stacks and Queues restrict access based on the order in which data was stored

    Collection ADT is the most basic ADT that provides ability to retrieve information non-reliant on where data is stored. 

    Assumptions for Collection 
        - Addition, Removal, and Retrieval of elements
        - Collection content access is based on equality of objects (using equals of methods)
        - Allow duplicate elements, do not allow null elements
        - add and remove operations return a boolean indicating success
                ** this ADT uses boolean instead of exceptions, like from previous ADTs

        
            public interface CollectionInterface
                {
                    boolean add(T element);
                    // Attempts to add element to collection. Returns true if successful, false otherwise

                    T get(T target);
                    // Returns an element e from collection such that e.equals(target).
                    // If no such element exists, return null.

                    boolean contains(T target);
                    // Returns true if this collection contains an element e such that
                    // e.equals(taget); otherwise returns false.

                    boolean remove(T target);
                    // Removes an element e from this collection such that e.equals(target)
                    // and returns true. if no such element exists, return false.

                    boolean isFull();
                    boolean isEmpty();
                    int size();
                }
