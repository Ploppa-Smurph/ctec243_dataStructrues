09.01 - Priority Queue and Heap

    Priority Queue is an abstract data type with and interesting accessing protocol - only highest-priority element can be accessed
        ** useful fo application that processes items by priority

    Priority Queue Interface
    enqueue
    dequeue
    isEmpty
    isFull
    size

    Priority Queue Implementations
    Unsorted List - dequeue requires search through entire list
    Array Based-Sorted List - enqueue expensive
    Sorted Linked List - enquees at O(N)
    Binary Search Tree - averages O(logN) for enqueue and dequeue 
    Heap - gurantees O(logN), even in worst case

    Heap - is an implementation of a Priority Queue that uses a 'binary tree' (not Binary Search Tree) and satisfies 2 properties:
        - Shape property - the tree must be a complete binary tree
        - Order property - for every node in the tree, the value stored in that node is greater than or equal to the value in each of its children
            -- there are 'minHeap' and 'maxHeap'

    Full binary tree - is totally full, ends up being somewhat triangular in layout because parent is small and it works down from left to right filling in the children, and acendents
